- (void) endFetchData

Closes the established long connection.

This method is used to close the long connection that has been established, ceasing the reception of marketing data pushed from the server. It should be called when the data is no longer needed to avoid unnecessary resource consumption and potential memory leaks.

Parameters

◆ startFetchDataWithCompletion: [1/2]

- (void) startFetchDataWithCompletion: (ThingSmartMarketingFetchDataCompletion)  completion

Initiates a request for marketing data and establishes a long connection.

This method is used to start a request for marketing data. Upon initiation, a long connection will be established to receive marketing data pushed from the server. Once this method is called, the completion callback will be executed to handle the initially fetched data.

Parameters
completionA ThingSmartMarketingFetchDataCompletion type completion callback. It is intended to handle the initial set of data that is fetched when the request is first made. It is not designed to be called repeatedly for data received over the established long connection.
