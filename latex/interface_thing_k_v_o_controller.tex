\doxysubsection{Detailed Description}
@abstract \mbox{\hyperlink{interface_thing_k_v_o_controller}{Thing\+KVOController}} makes Key-\/\+Value Observing simpler and safer. @discussion \mbox{\hyperlink{interface_thing_k_v_o_controller}{Thing\+KVOController}} adds support for handling key-\/value changes with blocks and custom actions, as well as the NSKey\+Value\+Observing callback. Notification will never message a deallocated observer. Observer removal never throws exceptions, and observers are removed implicitly on controller deallocation. \mbox{\hyperlink{interface_thing_k_v_o_controller}{Thing\+KVOController}} is also thread safe. When used in a concurrent environment, it protects observers from possible resurrection and avoids ensuing crash. By default, the controller maintains a strong reference to objects observed.
