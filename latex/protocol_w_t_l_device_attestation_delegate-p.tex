If -\/device\+Attestation\+Failed\+For\+Controller\+:opaque\+Device\+Handle\+:error\+: is implemented, then it will be called when device attestation fails, and the client can decide to continue or stop the commissioning.

If -\/device\+Attestation\+Completed\+For\+Controller\+:opaque\+Device\+Handle\+:attestation\+Device\+Info\+:error\+: is implemented, then it will always be called when device attestation completes. Notify the delegate when device attestation completed with device info for additional verification. If this callback is implemented, continue\+Commissioning\+Device on \mbox{\hyperlink{interface_w_t_l_device_controller}{WTLDevice\+Controller}} is expected to be called if commisioning should continue.

This allows the delegate to stop commissioning after examining the device info (DAC, PAI, CD).
